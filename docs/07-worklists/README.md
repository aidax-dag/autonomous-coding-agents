# 07 Worklists

> 목적: 전체 문서를 매번 읽지 않고, 미구현 기능 분석부터 구현 실행까지 바로 이어지는 작업 문서 허브

## Reading Order (Low-Token)

1. `00_WORKFLOW.md` - 토큰 최소 사용 실행 규칙
2. `10_STATUS_BASELINE.md` - 현재 구현/미구현 기준선
3. 필요한 트랙 문서 1개만 선택
4. 선택한 문서의 체크리스트를 구현
5. 구현 후 `60_DOC_SYNC_AND_TOKEN_POLICY.md` 기준으로 상태 업데이트

## Worklist Documents

- `20_RUNTIME_AND_CONNECTORS.md`
- `30_ANALYZER_AND_GENERATOR_IMPLEMENTATION.md`
- `40_BENCHMARK_AND_DEEPWORKER.md`
- `50_STUB_PRODUCTIONIZATION.md`
- `60_DOC_SYNC_AND_TOKEN_POLICY.md`

## Rule

- 한 번에 하나의 트랙 문서만 연다.
- 구현 전에는 해당 트랙 문서의 `Out of Scope`를 먼저 확인한다.
- 구현 완료 시 체크박스와 Evidence 섹션만 갱신한다.
