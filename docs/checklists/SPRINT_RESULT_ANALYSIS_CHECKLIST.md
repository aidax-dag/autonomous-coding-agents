# Sprint Result & Analysis Checklist

## 1. 결과정리

- [ ] 스프린트 목표 대비 달성률을 기록했다.
- [ ] 완료/미완료 티켓 목록을 정리했다.
- [ ] 주요 산출물 링크를 정리했다.

## 2. 성과 지표 분석

- [ ] 리드타임(티켓 생성->완료)을 측정했다.
- [ ] 완료량(스프린트 처리 티켓 수)을 측정했다.
- [ ] 변경 실패율/재작업률을 측정했다.
- [ ] 티켓당 토큰 사용량을 분석했다.
- [ ] Feature 재사용률을 분석했다.

## 3. 비용/효율 분석

- [ ] 반복 구현으로 인한 낭비를 식별했다.
- [ ] 재사용으로 절감된 시간/토큰을 추정했다.
- [ ] 다음 스프린트 최적화 포인트를 도출했다.

## 4. 이슈/리스크 회고

- [ ] 스프린트 중 발생한 이슈를 분류했다.
- [ ] 근본 원인과 예방 액션을 정의했다.
- [ ] 보안/운영 리스크를 업데이트했다.

## 5. 새로운 요구사항 업데이트

- [ ] 신규 요구사항을 작성했다.
- [ ] 기존 요구사항 변경/삭제/보류를 반영했다.
- [ ] 우선순위(P0/P1/P2)를 재정렬했다.
- [ ] 다음 스프린트 입력 티켓을 생성했다.

## 6. 승인

- [ ] 결과 분석 리뷰 승인 완료.
- [ ] 다음 스프린트 시작 승인 완료.
