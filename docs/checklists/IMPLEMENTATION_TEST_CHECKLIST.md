# Implementation & Test Checklist

## 1. 구현 품질

- [ ] 구현 범위가 승인된 스펙과 일치한다.
- [ ] 임시 코드/하드코딩/디버그 코드가 제거되었다.
- [ ] 예외/에러 처리 경로가 정의되어 있다.
- [ ] 로그/관찰성 정보가 필요한 수준으로 남아 있다.
- [ ] 문서/코드 링크가 티켓에 첨부되었다.

## 2. 보안 점검

- [ ] 입력값 검증이 적용되었다.
- [ ] 인증/인가 경계가 깨지지 않았다.
- [ ] 시크릿/토큰/민감정보 노출이 없다.
- [ ] 외부 연동(MCP/A2A/결제) 실패 시 안전한 처리 경로가 있다.
- [ ] 감사 가능한 이력(누가/언제/무엇)을 남긴다.

## 3. 테스트 점검

- [ ] 단위 테스트 실행/통과.
- [ ] 통합 테스트 실행/통과.
- [ ] 계약 테스트(인터페이스 호환성) 실행/통과.
- [ ] E2E 또는 시나리오 테스트 실행/통과.
- [ ] 실패 케이스 재현 및 해결 확인.

## 4. Feature 재사용 점검

- [ ] 기존 Feature 재사용 가능성을 먼저 검토했다.
- [ ] 신규 Feature 생성 시 라벨/옵션/제약을 기록했다.
- [ ] 버전 및 버전 이력 업데이트를 확인했다.
- [ ] usageCount/lastUsedAt 등 재사용 지표를 갱신했다(해당 시).

## 5. 티켓 완료 조건 점검

- [ ] 산출물 링크 기록 완료.
- [ ] 검증 체크리스트 완료.
- [ ] 리뷰어 승인 완료.
- [ ] 상태가 `reviewing`에서 `completed`로 정상 전이되었다.
