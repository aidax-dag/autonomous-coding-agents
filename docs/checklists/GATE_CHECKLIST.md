# Gate Checklist (Sprint Cycle)

## A. 구현 전 필수 게이트

### A1. 요구사항 게이트

- [ ] 배경/문제가 명확히 정의되었다.
- [ ] 목표/KPI가 측정 가능하다.
- [ ] 범위/비범위가 정의되었다.
- [ ] 제약사항이 기록되었다.
- [ ] 승인 완료 (`approved`).

### A2. 기능정의 게이트

- [ ] 기능 목록이 우선순위(P0/P1/P2)로 분류되었다.
- [ ] 의존성이 명시되었다.
- [ ] 작업 단위가 티켓으로 분해되었다.
- [ ] 승인 완료 (`approved`).

### A3. 설계 게이트

- [ ] 아키텍처 경계(도메인/모듈)가 명확하다.
- [ ] 상태 전이/실패 시나리오가 정의되었다.
- [ ] 보안 설계 포인트가 포함되었다.
- [ ] 변경 유연성(확장/교체 전략)이 정의되었다.
- [ ] 승인 완료 (`approved`).

### A4. 스펙정의 게이트

- [ ] API/스키마/입출력 명세가 완성되었다.
- [ ] 합격 조건이 명시되었다.
- [ ] 테스트 기준(단위/통합/계약/E2E)이 사전 확정되었다.
- [ ] 티켓 검증 체크리스트가 작성되었다.
- [ ] 승인 완료 (`approved`).

## B. 구현 이후 게이트

### B1. 구현 게이트

- [ ] 스펙 범위 내 구현 완료.
- [ ] 변경 파일/산출물 링크 기록 완료.
- [ ] 시작/종료 시각 기록 완료.
- [ ] 이슈/리스크/블로커 기록 완료.

### B2. 테스트 게이트

- [ ] 사전 정의한 필수 테스트를 실행했다.
- [ ] 실패 항목의 원인과 조치를 기록했다.
- [ ] 회귀 영향 범위를 점검했다.
- [ ] 합격 조건을 충족했다.

### B3. 리뷰 게이트

- [ ] 필수 리뷰어(Agent/Human) 검토 완료.
- [ ] `changes_requested` 항목이 모두 해소되었다.
- [ ] 리뷰 결론이 티켓에 반영되었다.

### B4. 문서 동기화 게이트

- [ ] 변경된 docs/ 파일이 있으면 `npm run docs:compact` 실행했다.
- [ ] 해당 워크리스트 트랙 문서의 Status/Evidence 섹션만 갱신했다.
- [ ] 불필요한 장문 회고 문서를 생성하지 않았다.

### B5. 종료 게이트

- [ ] 결과정리 및 분석 완료.
- [ ] 다음 요구사항 업데이트 완료.
- [ ] 티켓 상태를 `completed`로 전이했다.
