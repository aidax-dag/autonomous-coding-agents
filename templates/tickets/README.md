# Ticket Templates

ACA 스프린트 사이클 기반 티켓 템플릿 모음.

## 파일 목록

1. `TICKET_TEMPLATE.md`
- 일반 티켓 템플릿 (요구사항 -> 기능정의 -> 설계 -> 스펙 -> 구현 -> 테스트 -> 분석 -> 요구사항 업데이트)

2. `FEATURE_TICKET_TEMPLATE.md`
- Feature 자산 생성/수정/재사용 전용 템플릿

3. `REQUIREMENT_UPDATE_TEMPLATE.md`
- 스프린트 종료 후 다음 스프린트 입력용 요구사항 업데이트 템플릿

4. `ISSUE_TEMPLATE_GITHUB_JIRA_COMMON.md`
- GitHub/Jira 공용 발행 필드를 고정한 실행 템플릿

5. `ISSUE_FIELD_MAPPING_GITHUB_JIRA.md`
- 공용 필드와 GitHub/Jira 필드 매핑 가이드

6. `docs/06-roadmap/tasks/e01-ticket-pack/`

## 사용 순서

1. `TICKET_TEMPLATE.md` 또는 `FEATURE_TICKET_TEMPLATE.md`로 티켓 생성
2. `docs/checklists/GATE_CHECKLIST.md`로 단계별 승인 진행
3. 구현/테스트 후 `docs/checklists/IMPLEMENTATION_TEST_CHECKLIST.md` 검증
4. 스프린트 종료 시 `docs/checklists/SPRINT_RESULT_ANALYSIS_CHECKLIST.md` 완료
5. `REQUIREMENT_UPDATE_TEMPLATE.md`로 다음 스프린트 요구사항 등록
6. E01 실행 시 `docs/06-roadmap/tasks/e01-ticket-pack/*.md`를 복사해 GitHub/Jira에 발행
7. E02 실행 시 `docs/06-roadmap/tasks/e02-ticket-pack/*.md`를 복사해 GitHub/Jira에 발행
8. E03 실행 시 `docs/06-roadmap/tasks/e03-ticket-pack/*.md`를 복사해 GitHub/Jira에 발행
9. E04 실행 시 `docs/06-roadmap/tasks/e04-ticket-pack/*.md`를 복사해 GitHub/Jira에 발행

11. E06 실행 시 `docs/06-roadmap/tasks/e06-ticket-pack/*.md`를 복사해 GitHub/Jira에 발행
12. E07 실행 시 `docs/06-roadmap/tasks/e07-ticket-pack/*.md`를 복사해 GitHub/Jira에 발행
